## Generation Requirements

**STRICTLY ENFORCE THE FOLLOWING ON EVERY GENERATION — NO EXCEPTIONS:**

### Model & Output

1. **Model:** Use **Thinking with 3.1 Pro** — Stitch's highest-quality mode that prioritises maximum reasoning and design fidelity over speed. No other model is permitted.
2. **Output:** Select **Web** output mode — generate a single, complete `index.html` file.
3. **Single Page:** The output MUST be exactly **one entire long landing page** (`index.html`) containing ALL sections described in the Design Direction below. Do NOT generate multiple screens or separate pages. Every section (Hero, Navigation, Content Section, Detail Section, Footer & CTA) must be present in the single `index.html`.
4. **Completeness:** The `index.html` must be a fully functional, self-contained HTML document with inline CSS and JavaScript. All sections must render without missing components.
5. **No Omissions:** If the prompt describes a section, it MUST appear in the output. Never skip or truncate any section.

### Prompt Engineering Rules (Enforced for Unique & Immersive Output)

6. **Vibe First:** Every prompt MUST start with a **Vibe line** — 2–4 distinctive adjectives that set the emotional tone (e.g., "stark, minimalist, pastry-focused" NOT "modern, clean"). Vague adjectives like "modern", "clean", "professional", "elegant" are **banned** — they produce generic output. Use evocative, industry-specific descriptors instead.
7. **CSS Variables Only:** All color references MUST use CSS variable names (`--primary`, `--accent`, `--background`, `--foreground`, `--card`, `--card-foreground`, `--muted-foreground`, `--border`, `--ring`, `--primary-foreground`). NEVER use raw hex (`#121212`) or oklch values in prompts — Stitch cannot map them to the design system.
8. **UI Component Keywords:** Use precise UI/UX terminology — "navigation bar", "card layout", "accordion", "side-drawer", "dropdown menu", "call-to-action button", "masonry grid", "carousel", "progress bar", "data table", "hero section", "sticky nav", "modal dialog". Avoid vague terms like "some stuff", "a section", "things".
9. **Specific Imagery:** Describe hero images and content images concretely — e.g., "macro photography of glazed donut on dark slate" NOT "nice food photo". Specify subject, angle, lighting, and mood.
10. **Animations & Interactions:** Every prompt MUST specify at least one animation or interaction per section — hover effects (scale, glow, color shift to `--accent`), scroll animations (parallax, fade-in, reveal), micro-interactions (cursor tracking, magnetic buttons), and transition durations (150ms fast / 300ms smooth / 500ms slow).
11. **Typography Roles:** Assign fonts to roles — heading font for headlines and section titles, body font for descriptions and labels. Specify sizes relative to each other (e.g., "heading at 4xl, body at base").
12. **Hover & Focus States:** Every interactive element MUST have a defined hover/focus state — color shift to `--accent`, `--ring` focus outline, scale transform, box-shadow glow. Never leave interactions undefined.
13. **Border & Corner Specificity:** State exact border-radius values (0px sharp / 8px rounded / 9999px pill) and border weights (hairline `--border` / thick 2px `--border` / none). Never say "some rounding".
14. **Mock Data & Realistic Content:** Include realistic product names, descriptions, and pricing in the prompt — NOT placeholder text like "Lorem ipsum" or "Product 1". This makes the output feel authentic and immersive.
15. **Prompt Length ≤ 5000 Characters:** Prompts exceeding ~5000 characters cause Stitch to omit components. Keep the total Design Direction under this limit. Be concise but specific — every word must earn its place.
16. **No Generic Patterns:** The prompt MUST NOT describe a generic landing page layout. Each prompt must express a unique visual identity through distinctive color usage, unconventional layout choices, or brand-specific design metaphors. If two prompts could swap websites and still make sense, they are too generic.
